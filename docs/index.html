swiss-cheese · layered defense for Claude Code ·v0.3.0

Ship fewer defects with layered defense

Every quality gate — linting, tests, hooks, AI review, human review — is a slice of Swiss cheese: useful, but full of holes. swiss-cheese stacks those slices for Claude Code, cheaply, so a defect only ships when the holes in every slice line up.

You don't need perfect defenses. You need enough imperfect ones that the holes never align.

How it works

Defense-in-depth, engineered into the data structure

Not a prompt that asks Claude to be careful — deterministic mechanisms that fail closed, run before the LLM, and get sharper every session.

▤

Deterministic pre-LLM guards

run_guards.py scans the diff as data, never executing it — injection, secrets (with redaction), policy thresholds, and slopsquat / hallucinated deps. Blockers are enforced at commit time by a PreToolUse hook. Zero tokens.

◈

An unremovable review-lens floor

select_agents.py returns a rule-computed required set, separate from escalation_allowed. The boundary lives in the data structure, not in a “please don't remove” prompt. It's never empty for a real diff.

⊞

Independent parallel lenses

Read-only subagents — core, security, tests, performance, architecture, docs, staff — review the redacted diff in parallel. High-risk paths escalate architecture & staff to Opus. Every finding carries a verification field.

▦

Config v2 & fail-closed audit

Layers keyed by id with auto / comment / skip modes. A missing binary is skipped, never a silent pass. Findings stay active in the audit log until an explicit finding_dismissed line exists.

◲

Agents remember your project

Each lens carries memory: project, committed under .claude/agent-memory/, with an UPDATE / STALE / RESOLVED revalidation protocol — so every run is sharper than the last.

⌥

It never kills your session

Every internal guard error is exit 0 — a layer may have holes, but it fails open on itself and closed on the defect. The repository eats its own cooking: two CI slices guard every PR.

The stack

Native skills, not legacy commands

Consciously-invoked slash commands plus model-invoked domain knowledge that costs nothing until a task matches it.

/swiss-cheese:init
Probes the repo, detects how to run your tools (Makefile → package.json → composer → pyproject → …), proposes high-risk paths and a permissions.deny stanza, and writes config v2 — without ever editing your settings itself.
/swiss-cheese:intent
Reconstructs a ticket into intent, acceptance criteria, a test plan, scope guards, a risk class and an AI-disclosure block — then stops, writing no code. (Haiku.)
/swiss-cheese:review
diff snapshot → deterministic guards → lens selection (the unremovable floor) → independent read-only lenses on the redacted diff, in parallel → one ranked report.
/swiss-cheese:loop <task>
Implement → check layers → guards → review → fix → repeat, until green or the iteration budget runs out.
/swiss-cheese:pair
A numbered list of hard, break-it questions on your change. Sonnet, read-only — it can't write code.
/swiss-cheese:custom-layer
A five-step methodology for building a defense layer the catalog doesn't cover — name the failure mode, pick the cheapest mechanism, implement, analyze the new holes, test.
:layer · :status
:knowledge · :audit
Manage the stack, render it, wire task sources, and grade the knowledge layers.

Block a leak at commit — 0 tokens

A PreToolUse hook scans the staged diff as data. If a blocker lands at or above your threshold, the commit is stopped by exit 2 — no model, no tokens.

shell
$ git commit -m "add S3 uploader"

[swiss-cheese guards] commit blocked — findings at or above 'high':
  - [blocker] secrets src/config.py:12
      possible aws-access-key committed in diff
  - [high] injection CLAUDE.md
      modifies agent-control file — poisons
      downstream layers
Resolve these or set the guards layer to `comment`.

# commit aborted before the model ever woke up

Step by step

A real change, end to end

Ship one authenticated endpoint — GET /api/export, touching a high-risk auth/ path — with every slice stacked. Here's the whole run, from empty repo to a merge you trust.

1
/swiss-cheese:init

Set up the stack once

Two probes — repo_probe.py and runner_detector.py — read your languages, manifests and CI, then detect exactly how to run each tool (Makefile → package.json → pyproject → binaries). One round of questions, and it writes .swiss-cheese/config.json (schema v2). It never touches .claude/settings.json itself.

/swiss-cheese:init
probing repo…
detected  python · pytest · ruff · mypy · .github/workflows
runners   test → make test  lint → ruff check .  (high confidence)
high-risk src/auth/**, src/payments/**, db/migrations/**

? risk profile  ›  standard
? layers        ›  lint typecheck tests guards review docs human-review

✓ wrote .swiss-cheese/config.json (7 layers, block_at=high)
✓ appended governance section to CLAUDE.md
→ proposed a permissions.deny stanza (yours to paste)
2
/swiss-cheese:intent

Turn the ticket into a contract — before any code

A Haiku, read-only subagent reconstructs the ticket into intent, acceptance criteria, a test plan, scope guards, a risk class, and a paste-ready AI-disclosure block (which later satisfies the policy guard). It writes no code and stops.

/swiss-cheese:intent JIRA-482
Intent       expose account data as CSV, authenticated users only.
Acceptance   ☐ 200 + CSV for a valid session
             ☐ 401 for missing / expired token
             ☐ export scoped to the caller's own account
Test plan    happy path · expired token · cross-account attempt
Scope        touch src/auth/**, src/api/export.py only
Risk         high (auth path + PII in response)

— no code written. run /swiss-cheese:loop to implement —
3
/swiss-cheese:loop "JIRA-482 …"

Implement inside the loop

Implement → run the fast scripted layers in one call (check_layers.py --fast) → fix what the tail shows → repeat. ok is computed only from auto layers that failed; a missing binary is skipped, never a silent pass. One honest line per iteration.

/swiss-cheese:loop
plan → implement GET /api/export + tests

iter 1: lint ✅ tests ❌(2) → fixed → guards clean → review: 1 high
iter 2: lint ✅ tests ✅ → guards clean → review: clean ✅

converged — scripted layers pass, no blocker/high remaining
4
git commit

A guard catches the leak you didn't mean to ship

You hardcoded a token while debugging. At git commit, the deterministic guard scans the staged diff, redacts the secret into diff.redacted.patch, and blocks — spending zero tokens. Remove it, pull it from env, commit again.

shell
$ git commit -m "wire up export auth"
[swiss-cheese guards] commit blocked — at or above 'high':
  - [blocker] secrets src/api/export.py:8: github-pat in diff

$ git commit -m "wire up export auth"   # token → env
[main 9f2a1c7] wire up export auth   2 files changed
5
/swiss-cheese:review

The layered review escalates itself

Because the diff touches src/auth/**, the deterministic selector forces the staff lens into the required floor and escalates architecture & staff to Opus. The security lens finds a timing oracle a linter never could — with a concrete fix and a verification.

/swiss-cheese:review
floor  core security tests staff  → staff+arch on Opus

blocker · src/auth/session.py:47 (security)
  token checked with == — timing oracle
  fix  hmac.compare_digest(token, expected)

$ # apply fix, re-run
review: clean ✅  — 0 blocker, 0 high
6
git commit · human review

Green, with a human gate that fails closed

Scripted layers pass, guards are clean, the AI review is empty. What's left is the human-review comment layer and the AI-disclosure block in the PR. A finding is only ever retired when a real reason is written to the fail-closed audit log — nothing gets rubber-stamped by silence.

Get started

Two commands in Claude Code

Add the marketplace, install the plugin. If your environment can't fetch marketplaces from GitHub directly, clone first and point at the local path.

01 — from GitHub
claude code 
/plugin marketplace add jakubciszak/code-quality-confidence
/plugin install swiss-cheese@code-quality-confidence
02 — from a local clone
shell + claude code 
# clone anywhere on disk
git clone https://github.com/jakubciszak/code-quality-confidence.git

# then, in Claude Code:
/plugin marketplace add /path/to/code-quality-confidence
/plugin install swiss-cheese@code-quality-confidence

Then run /swiss-cheese:init to detect your toolchain and write config v2.
