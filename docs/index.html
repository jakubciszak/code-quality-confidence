Ship fewer defects with layered defense

Every quality gate — linting, tests, hooks, AI review, human review — is a slice of Swiss cheese: useful, but full of holes. swiss-cheese stacks those slices for Claude Code, cheaply, so a defect only ships when the holes in every slice line up.

You don't need perfect defenses. You need enough imperfect ones that the holes never align.

How it works

Defense-in-depth, engineered into the data structure

Not a prompt that asks Claude to be careful — deterministic mechanisms that fail closed, run before the LLM, and get sharper every session.

▤

Deterministic pre-LLM guards

run_guards.py scans the diff as data, never executing it — injection, secrets (with redaction), policy thresholds, and slopsquat / hallucinated deps. Blockers are enforced at commit time by a PreToolUse hook. Zero tokens.

◈

An unremovable review-lens floor

select_agents.py returns a rule-computed required set, separate from escalation_allowed. The boundary lives in the data structure, not in a “please don't remove” prompt. It's never empty for a real diff.

⊞

Independent parallel lenses

Read-only subagents — core, security, tests, performance, architecture, docs, staff — review the redacted diff in parallel. High-risk paths escalate architecture & staff to Opus. Every finding carries a verification field.

▦

Config v2 & fail-closed audit

Layers keyed by id with auto / comment / skip modes. A missing binary is skipped, never a silent pass. Findings stay active in the audit log until an explicit finding_dismissed line exists.

◲

Agents remember your project

Each lens carries memory: project, committed under .claude/agent-memory/, with an UPDATE / STALE / RESOLVED revalidation protocol — so every run is sharper than the last.

⌥

It never kills your session

Every internal guard error is exit 0 — a layer may have holes, but it fails open on itself and closed on the defect. The repository eats its own cooking: two CI slices guard every PR.

The stack

Native skills, not legacy commands

Consciously-invoked slash commands plus model-invoked domain knowledge that costs nothing until a task matches it.

/swiss-cheese:init
Probes the repo, detects how to run your tools (Makefile → package.json → composer → pyproject → …), proposes high-risk paths and a permissions.deny stanza, and writes config v2 — without ever editing your settings itself.
/swiss-cheese:intent
Reconstructs a ticket into intent, acceptance criteria, a test plan, scope guards, a risk class and an AI-disclosure block — then stops, writing no code. (Haiku.)
/swiss-cheese:review
diff snapshot → deterministic guards → lens selection (the unremovable floor) → independent read-only lenses on the redacted diff, in parallel → one ranked report.
/swiss-cheese:loop <task>
Implement → check layers → guards → review → fix → repeat, until green or the iteration budget runs out.
/swiss-cheese:pair
A numbered list of hard, break-it questions on your change. Sonnet, read-only — it can't write code.
/swiss-cheese:custom-layer
A five-step methodology for building a defense layer the catalog doesn't cover — name the failure mode, pick the cheapest mechanism, implement, analyze the new holes, test.
:layer · :status
:knowledge · :audit
Manage the stack, render it, wire task sources, and grade the knowledge layers.
Get started

Two commands in Claude Code

Add the marketplace, install the plugin. If your environment can't fetch marketplaces from GitHub directly, clone first and point at the local path.

01 — from GitHub
claude code 
/plugin marketplace add jakubciszak/code-quality-confidence
/plugin install swiss-cheese@code-quality-confidence
02 — from a local clone
shell + claude code 
# clone anywhere on disk
git clone https://github.com/jakubciszak/code-quality-confidence.git

# then, in Claude Code:
/plugin marketplace add /path/to/code-quality-confidence
/plugin install swiss-cheese@code-quality-confidence

Then run /swiss-cheese:init to detect your toolchain and write config v2.
